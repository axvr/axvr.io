A Better Website

I have recently redesigned my website from the ground up, and there are some things I would like to mention about why it was designed this way.

  • No JavaScript. Plain HTML and CSS static website.
  • No analytics or cookies.
  • No advertisements, the site is not trying to manipulate the visitors.
  • No clutter. The page layout should be simple, and easy to use and read.
  • Viewable on any device or browser. Although the site was built using CSS Grid a relatively new CSS technology, the site is still fully functional and viewable in any browser, and scales appropriately to fit the dimensions of any device while maintaining full ease of readability. It should even work perfectly with terminal web browsers
  • No dependence upon anything. Anything the site needs it is hosting itself. e.g. fonts (no reliance on Google)
  • Open Source. The site is licenced under the ISC licence, and the content under the CC-BY-4.0 licence. And it makes this clear to the visitors.
  • Simple static HTML page generator. I custom made the HTML page generator to be as simple as possible, and follow the Unix philosophy (it is also fully POSIX compliant - no GNU/Bash-isms).
  • Fully compatible with html readers (e.g. Pocket and Firefox reader view)
  • Consistent, the page should not change in a way the viewer was not expecting while navigating the site
  • Not HTML Hell. http://www.catb.org/~esr/html-hell.html
  • Works perfectly for those who refuse to use closed source / proprietary software, including JavaScript
  • Decentralised or able to move easily. My website can easily be moved to a different hosting provider, or be self hosted easily, because although I am using GitHub Pages to host the site, I am not using Jekell to generate the site.
  • RSS feed, all websites which contain a blog should provide an RSS feed for that blog.
  • SSL. Need I say more.
  • Try to prevent link rot. e.g. when referencing/linking to PDFs try to provide copies yourself, when the copyright allows it (or is contained within the document).
  • No pop ups. Why these are even still a thing I don't know.
  • Works with adblockers.
  • decent max width, for easy reading, don't do a wikipedia and span the content to the width of the screen, it becomes difficult to read
  • Little complexity
